The Story Behind PokeClaw

Gemma 4 launched on a Tuesday. By Thursday I had a working app that uses it to control a phone. Two all-nighters, one person, and a lot of Claude Code.

I searched for months before building this. Every open-source phone automation project I found either needs a cloud LLM, uses screenshot-based vision (fragile), or doesn't ship a working APK. None of them run inference locally with native tool calling against the live accessibility tree.

PokeClaw is a prototype built from scratch by a solo developer. It's not a polished consumer app. If it works on your device, amazing. If it breaks, issues are welcome.

The entire codebase is open. Read it, fork it, build on it. That's the point.

Frequently Asked Questions

What phones are supported?

Android 9+ with arm64 architecture. The 2B model needs about 4 GB free RAM. Budget phones work but expect ~45 second warmup on CPU. Flagships with ML chips (Pixel 8+, Galaxy S24, Dimensity 9200+) are significantly faster with GPU acceleration.

Is it really free? What's the catch?

No catch. Apache 2.0 license, fully open source. No ads, no data collection, no premium tier, no subscription. The entire codebase is on GitHub. You can build it from source yourself.

How is this different from Google Assistant?

Google Assistant works with a limited set of Google-integrated apps and needs internet. PokeClaw controls ANY app on your phone through Accessibility Services and works completely offline. It reads the actual UI elements on screen and figures out the taps itself.

Does it work offline?

Yes. After the one-time 2.6 GB model download, everything runs on your phone. You can block PokeClaw from the internet completely and it still works. Zero network calls in local mode.

Is my data safe?

The model runs entirely on your device. Your conversations, messages, and screen content never leave your phone. There is no server, no telemetry, no analytics in the app itself. The source code is open for you to verify.

Can I use cloud models too?

Yes. PokeClaw supports any OpenAI-compatible API (GPT-4o, Claude, Gemini, etc). Bring your own API key in Settings. Cloud mode is smarter for complex tasks. You can switch between local and cloud mid-conversation.

Samsung says it's malware?

That's a false positive from sideloading. Samsung flags any unsigned APK installed outside the Play Store. The app is fully open source. Zero data collection, zero network calls in local mode. You can build from source to verify.
